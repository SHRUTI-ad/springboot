AWS Overview

What is AWS?

AWS (Amazon Web Services) provides on-demand cloud computing platforms. Key services include EC2, S3, Lambda, and RDS.

What is Amazon S3?

Amazon S3 is a scalable, durable, and low-latency object storage service.

Use of Amazon S3

  • Storage of large amounts of data: Used for files, backups, media content, logs, and static assets.
  • Durability: Data is automatically replicated across multiple data centers to ensure availability.
  • Access from anywhere: Accessible over the internet to retrieve data globally.

What is Amazon EC2?

Amazon EC2 provides scalable compute capacity in the cloud, allowing you to launch virtual servers (instances) on-demand.

Use of Amazon EC2

  • Run applications: Host websites, process workloads, and run applications on virtual instances.
  • Flexible instance types: Choose from various types of instances based on computing needs.
  • Cost-effective: Pay only for the compute time you use, making it economical.

Key Features of Amazon EC2

  • On-demand Instances: Start and stop instances whenever needed.
  • Scalability: Scale instances up or down based on your requirements.
  • Elastic Load Balancing: Distribute traffic across multiple instances for high availability.

Git Overview

Git is a distributed version control system. Below are commonly used Git commands:

  • git init: Initializes a new Git repository in the current directory.
  • git clone <repository_url>: Creates a copy of a remote Git repository on your local machine.
  • git status: Shows the status of changes in your working directory and staging area.
  • git add <file>: Stages a specific file to be committed (e.g., git add index.html).
  • git commit -m "message": Commits staged changes with a message describing the changes.
  • git pull: Fetches and merges changes from the remote repository into your current branch.
  • git push: Pushes your local commits to the remote repository.
  • git branch: Lists all branches in your repository, highlighting the current branch.
  • git checkout <branch>: Switches to a different branch in your repository.
  • git log: Shows the commit history of the current branch, displaying commit IDs, author info, and messages.

Linux Overview

Linux is an open-source operating system. Here are 10 basic Linux commands:

  • ls: List files and directories in the current directory.
  • cd: Change the current directory (e.g., cd /home/user).
  • mkdir: Create a new directory (e.g., mkdir new_folder).
  • rm: Remove files or directories (e.g., rm file.txt, rm -r folder/).
  • cp: Copy files or directories (e.g., cp file1.txt file2.txt).
  • mv: Move or rename files or directories (e.g., mv oldname.txt newname.txt).
  • touch: Create an empty file (e.g., touch newfile.txt).
  • cat: View the contents of a file (e.g., cat file.txt).
  • echo: Display a message or create a file with text (e.g., echo "Hello, World!" > file.txt).
  • pwd: Print the current working directory.

SQL Overview

SQL (Structured Query Language) is used to manage and manipulate relational databases. Below are common SQL operations:

  • DDL: Data Definition Language (CREATE, ALTER, DROP).
  • DML: Data Manipulation Language (INSERT, UPDATE, DELETE).
  • DQL: Data Query Language (SELECT).
  • DCL: Data Control Language (GRANT, REVOKE).

SQL Joins

Joins are used to combine rows from two or more tables based on a related column.

  • INNER JOIN: Returns records with matching values in both tables.
  • LEFT JOIN: Returns all records from the left table, and the matched records from the right table.
  • RIGHT JOIN: Returns all records from the right table, and the matched records from the left table.
  • FULL JOIN: Returns all records when there is a match in either left or right table.

Examples

DDL Query:

CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    Department VARCHAR(50)
);

DML Query:

INSERT INTO Employees (EmployeeID, FirstName, LastName, Department)
VALUES (1, 'John', 'Doe', 'HR');

DQL Query:

SELECT * FROM Employees;

DCL Query:

GRANT SELECT ON Employees TO 'username';

GROUP BY and ORDER BY

These clauses are used to organize and sort query results:

  • GROUP BY: Groups rows sharing a property into summary rows.
  • ORDER BY: Sorts the results in ascending (ASC) or descending (DESC) order.

Example:

SELECT Department, COUNT(*) AS EmployeeCount
FROM Employees
GROUP BY Department
ORDER BY EmployeeCount DESC;

Docker Overview

What is Docker?

Docker is an open-source platform designed to automate the deployment, scaling, and management of applications using containerization. Containers are lightweight, standalone, and executable units that package the application and its dependencies together, ensuring consistent performance across environments.

Key Features of Docker

  • Lightweight: Containers share the host OS kernel, making them faster and more efficient than virtual machines.
  • Portability: "Build once, run anywhere" â€” Docker containers can run on any system with Docker installed.
  • Scalability: Easily scale applications across multiple containers.
  • Isolation: Applications run in isolated environments, reducing dependency conflicts.

Common Docker Components

  • Docker Engine: The core service for creating and running containers.
  • Docker Image: A pre-configured template to create containers.
  • Docker Container: A running instance of a Docker image.
  • Dockerfile: A script that defines how to build a Docker image.
  • Docker Hub: A cloud-based registry for sharing Docker images.

Below are some common Docker commands:

  • docker build: Build an image from a Dockerfile.
  • docker run: Run a container.
  • docker ps: List running containers.

Jenkins Overview

What is Jenkins?

Jenkins is an open-source automation tool written in Java, used to automate tasks like building, testing, and deploying software in a Continuous Integration and Continuous Delivery (CI/CD) pipeline. It simplifies software development by integrating various stages of the pipeline into a single, automated process.

Key Features of Jenkins

  • Free and Open-Source: Completely free to use with a large, active community.
  • Extensibility: Jenkins has over 1,000 plugins to support building, deploying, and automating projects.
  • Distributed Builds: Jenkins can distribute tasks across multiple machines to speed up the build process.

Here are a few basic Jenkins commands:

  • java -jar jenkins.war: Start Jenkins.
  • Jenkins CLI: Interact with Jenkins through command-line tools.
  • jenkins-cli.jar build: Trigger builds from the command line.
